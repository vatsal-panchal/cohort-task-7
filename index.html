    Attributes vs Properties

    An attribute is the value written in HTML. A property is the live JavaScript object value — they can drift apart.

    Why they differ

    getAttribute('data-status') always returns the exact string from the HTML markup, even after JavaScript changes things elsewhere. element.dataset.status reads the live DOM property, reflecting the current state instantly. For built-ins like input.value, the attribute holds the original default while the property holds what the user is typing right now — that's the core distinction this assignment is testing.

    Difference

    Bubbling is the default phase: the event fires on the target first, then travels outward — Child → Parent → Grandparent. Capturing is the opposite phase, enabled by passing true as the third argument to addEventListener — it travels inward before the target is reached: Grandparent → Parent → Child.

    DOM + CSSOM — the visible boxes painted to the screen

    The browser parses HTML into a DOM tree describing structure, and CSS into a CSSOM tree describing style. It combines the two into a Render Tree containing only the nodes that are actually visible, then performs Layout to calculate positions and Paint to draw pixels — the same pipeline this app's task cards go through every time one is added.
